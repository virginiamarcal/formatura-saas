## 📋 Technical Details

### Database Schema

**Table: event_attendees** (NEW)
```sql
- id (UUID, PK)
- event_id (UUID, FK → events.id, ON DELETE CASCADE)
- user_id (UUID, FK → auth.users.id, ON DELETE CASCADE)
- email (VARCHAR 255, NOT NULL)
- name (VARCHAR 255, NOT NULL)
- role (ENUM: admin, member, guest, DEFAULT member)
- invited_at (TIMESTAMP, DEFAULT now())
- joined_at (TIMESTAMP, NULL)
- status (ENUM: pending, accepted, declined, DEFAULT pending)
- created_at (TIMESTAMP, DEFAULT now())
- updated_at (TIMESTAMP, DEFAULT now())

UNIQUE (event_id, user_id)
UNIQUE (event_id, email)
```

### RLS Policies

**event_attendees table:**
- Admin of event can SELECT/INSERT/UPDATE attendees for their event
- Policy: `(SELECT EXISTS(SELECT 1 FROM events WHERE id=event_id AND admin_id=auth.uid()))`
- Attendees can SELECT their own record
- Policy: `(user_id = auth.uid() OR (event_id in (SELECT id FROM events WHERE admin_id = auth.uid())))`

### Triggers

**event_attendees:**
- `trigger_updated_at_event_attendees` — Auto-update `updated_at` on row change
