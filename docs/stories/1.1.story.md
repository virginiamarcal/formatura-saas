## 📖 User Story

**As an** admin
**I want to** create a reusable proposal template for the event
**So that** I can standardize formatting and reduce manual work

---

## ✅ Acceptance Criteria

- [ ] Admin can navigate to "Proposals" section from the dashboard
- [ ] "Create Proposal" button is visible and clickable
- [ ] Admin can create new proposal with the following fields:
  - [ ] Title (required, string, max 255 chars)
  - [ ] Description (optional, text area)
  - [ ] Version (auto-set to 1.0, editable)
- [ ] Admin can add dynamic sections to proposal:
  - [ ] Section name (required, max 100 chars)
  - [ ] Section content (required, rich text editor)
  - [ ] Field type (enum: text, number, date, currency, textarea)
  - [ ] Order sections via drag-and-drop
  - [ ] Edit/delete sections
- [ ] Proposal sections are editable/locked based on business rules
- [ ] Proposal saved with status='draft'
- [ ] Admin can preview rendered proposal in HTML format
- [ ] Success message displayed: "Proposal template created successfully"
- [ ] Proposal appears in proposals list

## 📋 Technical Details

### Database Schema (by Dara)

**Table: proposals**
```sql
- id (UUID, PK)
- event_id (UUID, FK → events.id)
- title (VARCHAR 255, NOT NULL)
- description (TEXT, NULL)
- version (DECIMAL 2,1, DEFAULT 1.0)
- status (ENUM: draft, sent, approved, rejected, DEFAULT draft)
- created_at (TIMESTAMP, DEFAULT now())
- updated_at (TIMESTAMP, DEFAULT now())
- created_by (UUID, FK → auth.users.id)
```

**Table: proposal_sections**
```sql
- id (UUID, PK)
- proposal_id (UUID, FK → proposals.id, ON DELETE CASCADE)
- section_name (VARCHAR 100, NOT NULL)
- content (TEXT, NOT NULL)
- field_type (ENUM: text, number, date, currency, textarea)
- section_order (INTEGER, NOT NULL)
- is_editable (BOOLEAN, DEFAULT true)
- created_at (TIMESTAMP, DEFAULT now())
- updated_at (TIMESTAMP, DEFAULT now())
```

### RLS Policies

**proposals table:**
- Admin of event can INSERT/UPDATE/SELECT their own proposals
- Policy: `(auth.uid() = created_by AND event.admin_id = auth.uid())`

**proposal_sections table:**
- Same as proposals (inherit via FK relationship)

### Triggers

**proposals:**
- `trigger_updated_at_proposals` — Auto-update `updated_at` on row change

**proposal_sections:**
- `trigger_updated_at_proposal_sections` — Auto-update `updated_at` on row change

## 🎯 Implementation Steps

### Phase 1: Backend API (Express + Supabase)

#### 1.1.1 Create API Endpoint: POST /api/proposals ✅

**Status:** COMPLETED

**Endpoint:** `POST /api/proposals`

**Request Body:**
```json
{
  "event_id": "uuid",
  "title": "Annual Gala Proposal",
  "description": "Proposal for the 2026 Annual Gala",
  "version": 1.0,
  "sections": [
    {
      "section_name": "Event Overview",
      "content": "<p>Event details here...</p>",
      "field_type": "text",
      "section_order": 1
    }
  ]
}
```

**Response (201 Created):**
```json
{
  "id": "uuid",
  "event_id": "uuid",
  "title": "Annual Gala Proposal",
  "description": "Proposal for the 2026 Annual Gala",
  "version": 1.0,
  "status": "draft",
  "created_at": "2026-03-03T10:00:00Z",
  "updated_at": "2026-03-03T10:00:00Z",
  "sections": [
    {
      "id": "uuid",
      "section_name": "Event Overview",
      "content": "<p>Event details here...</p>",
      "field_type": "text",
      "section_order": 1
    }
  ]
}
```

**Error Handling:**
- 400: Missing required fields (title, event_id)
- 401: Unauthorized (not authenticated)
- 403: Forbidden (not admin of event)
- 409: Event not found
- 500: Database error

**Validation Rules:**
- title: max 255 chars, non-empty
- description: max 5000 chars
- version: must be number >= 1.0
- sections: max 50 sections per proposal
- section_name: max 100 chars, non-empty
- content: max 10000 chars per section
