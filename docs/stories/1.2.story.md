## 📖 User Story

**As an** admin
**I want to** send a proposal to committee members for review and approval
**So that** I can collect feedback and track approval status

---

## ✅ Acceptance Criteria

- [ ] Admin can select a draft proposal from proposals list
- [ ] "Send Proposal" button is visible and clickable (only for draft proposals)
- [ ] Admin can add committee members to send proposal to:
  - [ ] Search/select members from event attendees list
  - [ ] Add multiple recipients
  - [ ] Remove recipients before sending
- [ ] Admin can customize email message (optional)
- [ ] Send proposal button shows confirmation dialog
- [ ] Proposal status changes from 'draft' to 'sent' after sending
- [ ] Email sent to each committee member with:
  - [ ] Proposal preview/link to view
  - [ ] Customized message from admin
  - [ ] Request for feedback/approval
- [ ] Admin receives email confirmation of send
- [ ] Proposal appears in "Sent Proposals" view with recipient list
- [ ] System tracks send timestamp and recipient list
- [ ] Admin can resend proposal to specific members (optional)
- [ ] Success message displayed: "Proposal sent to {count} members"

## 📋 Technical Details

### Database Schema (by Dara)

**Table: proposal_sends** (NEW)
```sql
- id (UUID, PK)
- proposal_id (UUID, FK → proposals.id, ON DELETE CASCADE)
- sent_at (TIMESTAMP, DEFAULT now())
- sent_by (UUID, FK → auth.users.id)
- recipient_list (JSONB, array of {user_id, email, status, sent_date})
  Example: [
    { "user_id": "uuid", "email": "member@example.com", "status": "pending", "sent_date": "2026-03-03T10:00:00Z" },
    { "user_id": "uuid", "email": "member2@example.com", "status": "viewed", "sent_date": "2026-03-03T10:05:00Z" }
  ]
- custom_message (TEXT, NULL)
- created_at (TIMESTAMP, DEFAULT now())
- updated_at (TIMESTAMP, DEFAULT now())
```

**Table: proposal_feedback** (NEW - for Story 1.3)
```sql
- id (UUID, PK)
- proposal_id (UUID, FK → proposals.id)
- proposal_send_id (UUID, FK → proposal_sends.id)
- reviewer_id (UUID, FK → auth.users.id)
- status (ENUM: pending, approved, rejected, needs_revision)
- feedback_text (TEXT, NULL)
- updated_at (TIMESTAMP, DEFAULT now())
```

**Table: proposals (MODIFIED)**
- Add column: `status` updated to support 'sent', 'approved', 'rejected' states
- Add column: `last_sent_at` (TIMESTAMP, NULL) — tracks most recent send

### RLS Policies

**proposal_sends table:**
- Admin of event can INSERT/SELECT their own proposal sends
- Policy: `(auth.uid() = sent_by AND proposal.event_id = auth.uid())`

**proposal_feedback table:**
- Event admin can SELECT all feedback for their proposals
- Committee members can INSERT/UPDATE their own feedback
- Policy: `(auth.uid() = reviewer_id OR (auth.uid() = created_by OF parent proposal))`

### Triggers

**proposal_sends:**
- `trigger_updated_at_proposal_sends` — Auto-update `updated_at` on row change
- `trigger_update_proposal_status_to_sent` — Auto-update proposals.status = 'sent' when record inserted

**proposals:**
- `trigger_update_last_sent_at` — Auto-update `last_sent_at` when proposal_sends inserted

### Email Service Integration

**Service:** Supabase Email / SendGrid (configure in .env)

**Email Template Variables:**
- `{admin_name}` — Name of admin sending
- `{proposal_title}` — Title of proposal
- `{proposal_preview}` — First 200 chars of proposal content
- `{proposal_link}` — Deep link to view full proposal
- `{custom_message}` — Optional message from admin
- `{event_name}` — Name of event

#### 1.2.2 Create API Endpoint: POST /api/proposals/:id/send ✅

**Status:** PENDING

**Endpoint:** `POST /api/proposals/:id/send`

**Request Body:**
```json
{
  "recipient_user_ids": ["uuid1", "uuid2", "uuid3"],
  "custom_message": "Please review and provide feedback by Friday"
}
```

**Response (200 OK):**
```json
{
  "success": true,
  "proposal_id": "uuid",
  "send_id": "uuid",
  "recipients_count": 3,
  "emails_sent": 3,
  "failed_emails": [],
  "sent_at": "2026-03-03T10:00:00Z",
  "message": "Proposal sent to 3 members"
}
```

**Error Handling:**
- 400: Missing recipient_user_ids
- 400: Proposal not in draft status
- 401: Unauthorized (not authenticated)
- 403: Forbidden (not admin of event)
- 404: Proposal not found
- 409: Invalid recipient (not in event)
- 500: Email service failure

**Validation Rules:**
- recipient_user_ids: non-empty array, all must exist in event attendees
- custom_message: max 1000 chars
- Proposal status must be 'draft'
